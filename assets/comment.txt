1. flutter 란 
Google에서 개발한 오픈소스 UI 소프트웨어 개발 키트(SDK)

2. Flutter의 주요 특징
- 단일 코드베이스 (dart)
- 풍부한 위젯 (Material Design , Cupertino 위젯, 커스터마이징 가능한 위젯) 
- 핫 리로드 (Hot Reload) 실시간으로 코드 변경 사항을 반영

3. Flutter의 아키텍처
 Dart 언어 - 객체 지향적이며, 빠른 성능 제공
 모든 UI 요소은 위젯으로 구성, 
 선언적 UI 접근 방식
 Skia를 사용하여 고성능 그래픽 렌더링 (일관된 UI 표현 가능)

 4. Flutter의 장점 , 단점
  - 장점 : 
    단일 코드베이스로 여러 플랫폼 지원
    네이티브 컴파일 및 Skia 렌더링 엔진으로 높은 성능 제공
    풍부한 위젯 라이브러리, Material 및 Cupertino 지원, 일관된 UI 제공
    활발한 커뮤니티와 풍부한 라이브러리 (https://pub.dev/)
    주요 플랫폼(모바일, 웹, 데스크탑) 지원

  - 단점 : 
    Dart 언어 학습 필요, 플랫폼 특정 기능 접근 시 추가 작업 필요
    매우 복잡한 UI나 애니메이션에서 성능 저하 가능 (Lottie , rive 라이브러리를 사용하여 애니메이션을 구현할 수 있음)
    일부 네이티브 기능 제한, 최신 네이티브 API 지원 지연
    초기 바이너리 크기 증가
* 가장큰 단점 
    플러터로 개발하면 다른 플랫폼에서 사용할 수 없다.
